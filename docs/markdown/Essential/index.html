    Subset: essential¶

    Subset of properties that are considered to be essential for the functionality of GHGA.

    URI: GHGA:essential

    Classes¶

    Mixins¶

    Slots¶

    • affiliation - The Institution(s) associated with an entity.
    • age - Age of an individual.
    • alias - The alias for an entity.
    • analysis process➞has input - The input data File entities used in the Analysis Process.
    • analysis➞alias - An alias uniquely identifying this Analysis entitiy.
    • analysis➞description - Describing how an Analysis was carried out. (e.g.: computational tools, settings, etc.).
    • analysis➞has input - The input data File entities used in the Analysis.
    • analysis➞has study - The Study entity associated with this Analysis.
    • analysis➞reference chromosome
    • analysis➞reference genome
    • analysis➞type - The type of the Analysis. Either Reference Alignment (BAM) or Sequence Variation (VCF)
    • biospecimen➞alias - The alias for an entity.
    • biospecimen➞has disease - The Disease entity that is associated with the Individual. Typically, a concept from Mondo Disease Ontology. For example, 'MONDO:0005267' indicates that the Individual suffers from 'Heart Disease'.
    • biospecimen➞has individual - The Individual entity from which this Biospecimen was derived.
    • biospecimen➞has phenotypic feature - The Phenotypic Feature entity that is associated with the Individual. Typically, a concept from Human Phenotype Ontology. For example, 'HP:0100244' indicates that the Individual exhibits 'Fibrosarcoma' as one of its phenotype.
    • biospecimen➞type - The type of Biospecimen.
    • checksum - A computed value which depends on the contents of a block of data and which is transmitted or stored along with the data in order to detect corruption of the data. The receiving system recomputes the checksum based upon the received data and compares this value with the one sent with the data. If the two values are the same, the receiver has some confidence that the data was received correctly.
    • checksum type - The type of algorithm used to generate the checksum of a file.
    • cohort➞has individual - One or more Individuals that collectively define this Cohort.
    • data access committee➞alias - The alias for an entity.
    • data access committee➞description - A description for the Data Access Committee.
    • data access committee➞main contact - The main contact for the Data Access Committee.
    • data access committee➞name - The name for the Data Access Committee.
    • data access policy➞alias - The alias for an entity.
    • data access policy➞description - A short description for the Data Access Policy.
    • data access policy➞has data access committee - The Data Access Committee linked to this policy.
    • data access policy➞name - A name for the Data Access Policy.
    • data access policy➞policy text - The terms of data use and policy verbiage should be captured here.
    • dataset➞alias - The alias for an entity.
    • dataset➞description
    • dataset➞has analysis - One or more Analysis entities that are referenced by this Dataset.
    • dataset➞has data access policy - The Data Access Policy that applies to this Dataset.
    • dataset➞has experiment - One or more Experiment entities that are referenced by this Dataset.
    • dataset➞has file - One or more File entities that collectively are part of this Dataset.
    • dataset➞has sample - One or more Sample entities that are referenced by this Dataset.
    • dataset➞has study - One or more Study entities that are referenced by this Dataset.
    • dataset➞title - A title for the submitted Dataset.
    • dataset➞type - The type of a dataset.
    • description - Description of an entity.
    • experiment process➞has input - The input to the Experiment Process. Usually a Sample entity.
    • experiment process➞has protocol - The Protocol entity used by this Experiment Process.
    • experiment process➞title - A descriptive title that explains the step(s) involved in performing the experiment leading up to the sequencing of the sample and generation of raw data from the instrument. (eg: Sample extraction -> Target Enrichment)
    • experiment process➞type - The type of experiment process.
    • experiment➞alias - The alias for an entity.
    • experiment➞description - A detailed description of the Experiment.
    • experiment➞has experiment process - One or more Experiment Processes entities associated with this Experiment.
    • experiment➞has file - One or more Files entities that are generated as output of this Experiment.
    • experiment➞has protocol - One or more Protocol entities associated with this Experiment.
    • experiment➞has sample - The Sample entity associated with this Experiment.
    • experiment➞has study - The Study entity associated with this Experiment.
    • experiment➞title - Name for the experiment (eg: GHGAE_PBMC_RNAseq).
    • experiment➞type - The type of Experiment.
    • family➞has individual - One or more Individuals that collectively define this Family.
    • file➞alias - The alias for an entity.
    • file➞checksum
    • file➞checksum type
    • file➞format
    • file➞name - The given filename.
    • format - The format of the file: BAM, SAM, CRAM, BAI, etc.
    • has analysis - The analysis associated with an entity.
    • has data access committee - Data Access Committee associated with an entity.
    • has data access policy - Data Access Policy associated with an entity.
    • has disease - Disease concept that the entity is associated with.
    • has experiment - The experiment associated with an entity.
    • has experiment process
    • has file - The file associated with an entity.
    • has individual - The subject/individual associated with an entity.
    • has input - The input to a Planned Process.
    • has phenotypic feature - Phenotypic feature concept that the entity is associated with.
    • has protocol - The protocol associated with an entity.
    • has sample - The sample associated with an entity.
    • has study - The study associated with an entity.
    • id - An identifier that uniquely represents an entity.
    • individual➞age
    • individual➞alias - The alias for an entity.
    • individual➞has disease - The Disease entity that is associated with this Biospecimen at the time of retrieval from the organism. Typically, a concept from Mondo Disease Ontology. For example, 'MONDO:0003742' indicates that the Individual - from which the Biospecimen was extracted from - suffers from 'Heart Fibrosarcoma'.
    • individual➞has file - Additional/supplementary files associated with an Individual. Typically, a phenopacket or pedigree file.
    • individual➞has phenotypic feature - The Phenotypic Feature entity that is associated with this Biospecimen at the time of retrieval from the organism. Typically, a concept from Human Phenotype Ontology. For example, 'HP:0100244' indicates that the Individual - from which the Biospecimen was extracted from - exhibits 'Fibrosarcoma' as one of its phenotype.
    • individual➞sex
    • individual➞vital status - Last known Vital Status of an Individual.
    • instrument model - The name and model of the technology platform used to perform sequencing.
    • library layout - Describe whether the library was sequenced in single-end (forward or reverse) or paired-end mode
    • library name - A short name identifying the library to potential users. The same name may refer to multiple versions of the same continually updated library.
    • library preparation - The general method for sequencing library preparation (e.g. KAPA PCR-free).
    • library preparation protocol➞alias
    • library preparation protocol➞description - Description about how a sequencing library was prepared (eg: Library construction method).
    • library preparation protocol➞library layout
    • library preparation protocol➞library name
    • library preparation protocol➞library preparation
    • library preparation protocol➞library selection
    • library preparation protocol➞library type
    • library selection - Whether any method was used to select for or against, enrich, or screen the material being sequenced. Library Selection method (e.g. random, PCA, cDNA, etc )
    • library type - Describe the level of omics analysis (eg: Metagenome, transcriptome, etc)
    • main contact - The person who is the main contact for a committee.
    • name - The name for an entity.
    • named thing➞alias - The alias (alternate identifier) for an entity.
    • named thing➞id - The internal unique identifier for an entity.
    • ontology class mixin➞description - The description or definition of an ontology class.
    • ontology class mixin➞name - The name or label (rdfs:label) of an ontology class.
    • policy text - The complete text for the Data Access Policy.
    • project➞alias - The alias for an entity.
    • project➞description - Short textual description of the project.
    • project➞title - Comprehensive title for the project.
    • protocol➞alias - The alias for an entity.
    • protocol➞description - Detailed description of the Protocol.
    • protocol➞has file - A document that describes the Protocol.
    • protocol➞name - Name of the protocol (eg: Sample extraction_PCR amplification).
    • protocol➞type - Type of the protocol (eg: Target enrichment).
    • publication➞alias - The alias for an entity.
    • publication➞id - An identifier that uniquely represents an entity.
    • publication➞title - The title for the Publication.
    • reference chromosome - The reference chromosome used for this Analysis.
    • reference genome - A published genetic sequence that is used as a reference sequence against which other sequences are compared. Reference genome(s) or annotation(s) used for prior analyses (eg: GRCh38.p13).
    • sample➞alias - The alias for an entity.
    • sample➞description - Short textual description of the sample (How the sample was collected, sample source, protocol followed for processing the sample etc).
    • sample➞has individual - The Individual from which this Sample was derived from.
    • sample➞name - Name of the sample (eg:GHGAS_Blood_Sample1 or GHGAS_PBMC_RNAseq_S1).
    • sample➞type - The type of sample.
    • sequencing protocol➞alias
    • sequencing protocol➞description - Description about the sequencing protocol (eg: mRNA-seq,Whole exome long-read sequencing etc).
    • sequencing protocol➞instrument model
    • sequencing protocol➞type - Name of the library preparation protocol (eg: mRNA-seq,Whole exome long-read sequencing etc).
    • sex - The assemblage of physical properties or qualities by which male is distinguished from female; the physical difference between male and female; the distinguishing peculiarity of male or female.
    • study➞affiliation
    • study➞alias - The alias for an entity.
    • study➞description - A detailed description (abstract) that describes the goals of this Study.
    • study➞has file - Additional/supplementary files associated with a Study.
    • study➞title - A comprehensive title for the study.
    • study➞type - The type of Study. For example, 'Cancer Genomics', 'Epigenetics', 'Exome Sequencing'.
    • submission➞affiliation - Institution/Center/Data Hub that is providing this submission.

    • submission➞has data access policy - The Data Access Policy that applies to Dataset in this submission.
    • submission➞has experiment - Information about one or more Experiment entities associated with this submission.
    • submission➞has file - Information about one or more File entities associated with this submission.
    • submission➞has individual - Information about one or more Individual entities associated with this submission.
    • submission➞has protocol - One or more Protocol entities associated with this Submission.
    • submission➞has sample - Information about one or more Sample entities associated with this submission.
    • submission➞has study - Information about a Study entities associated with this submission.
    • submission➞id - A internal unique identifier for the Submission.
    • target regions - Subset of genes or specific regions of the genome, which are most likely to be involved in the phenotype under study.
    • tissue - An anatomical structure consisting of similarly specialized cells and intercellular matrix, aggregated according to genetically determined spatial relationships, performing a specific function.
    • title - The title that describes an entity.
    • type - The type of an entity.
    • vital status - The state or condition of being living or deceased; also includes the case where the vital status is unknown.
